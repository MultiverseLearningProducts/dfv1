Speaker notes editor 1.0

How it works

The plugin works by taking all the elements in a normal reveal.js presentation and adding in extra code to them, this generates all the functions used to run the plugin. The addition of this code happens as the presentation is loaded in the browser. When you save edits they are sent to your computer's memory, specifically the part that is accessed by your browser.
When you load a presentation that has been editted, The plugin finds that stored data in your browser memory, and adds it in to the presentation when it loads. This flow chart roughly shows the processes that are going on when the editor plugin is used.

Limitations

You must close the speaker notes and refresh after an editing session

It helps in understanding the limitations/requirements of the plugin. Because it requires a page load to start up th whole process, when you have saved your edits you MUST close down the editor window the speaker notes and then refresh the presentation. Otherwise your edits will not be loaded properly and won't appear in the speaker notes. When you click the save button the editor window will close itself but you MUST close speaker ntoes and refresh.

Edits are only recoverable on the browser that they were made

The second limitation of the plugin is that because it is linked to browser memory, you cannot share your edits between different browsers. If you edit in chrome the changes you made will only appear in chrome.

edits are personal

When you make edits they are saved to your machine ONLY, this means that your edits cannot influence the centrally approved notes or any other user's notes.

Accessing the editor mode

First access the speaker notes on your presentation by pressing 'S'.
In the notes section click the edit mode button to open a new editor session.
The editor window should track to wherever you are in the speaker notes, if this doesn't happen simply move the peaker notes 1 slide to the left or right using the arrow keys and the editor will track to that location.

Once you have started the editor mode you will have a window that looks like this. You may get an alert telling you that the popup is being blocked, you can remedy this by copying the domain name that is shown in the alert to your browsers accepted popup list.

Button functions explained

There are 5 basic features you will use

  • Save and close: Saves your changes and stores them in your browser memory, ready to be loaded when speaker notes is re-opened
  • Reset notes: Wipes all previously edited notes on THIS presentation, once you click save and close this will be permanent, to undo this (if you haven't saved yet) close down editor and speaker notes and then refresh the presentation
  • Reset reminders: Removes all reminders set on THIS presentation, once you click save and close this will be permanent, to undo this (if you haven't saved yet) close down editor and speaker notes and then refresh the presentation
  • Clear presentation: Clears all additions to THIS presentation, once you click save and close this will be permanent, to undo this (if you haven't saved yet) close down editor and speaker notes and then refresh the presentation
  • Tracker dots: You may have noticed there are no slide numbers in the editor window, it automatically tracks the speaker notes window and keeps the current slide at the bottom of the window.

How to edit the notes

To edit text open the editor window and select the pale blue sections you can type freely in these sections and linebreaks and formatting will be saved. The notes preset in the presentation are not editable.

Adding a reminder

To add a reminder in your session click the add reminder button in the editor window, you will be prompted to input a text reminder for yourself, when you run the presentation your reminder will pop up on your speaker notes and will have to be dismissed before your can proceed. These are good for key reminders, and must complete actions.

When you click the reminder button you will be prompted to add text for your reminder cancelling or leaving this box blank will not add a reminder to the presentation.

Making your edits appear in your speaker notes

To make your edits appear on your next session you must use close down the speaker notes and refresh the lesson

The skull and questionmark

Double clicking the skull will start a process to destroy all editor data saved on your browser this means all presentations! The only reason for using this would be to clear memory if it is full. Don't worry if you accidentally click this as you must confirm that you want all data deleted before it will proceed

The question mark will open up this readme page, you already worked that out

Adding speaker notes editor to GITHUB

To add to GITHUB repo take the editor dependencies and add it to the root of your repo, then add a script file with src= path to editor.js to any presentation you want the plugin to be accessible for.
